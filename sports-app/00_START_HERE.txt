EXAMPLE: CONVERTING A SCREEN
═══════════════════════════════════════════════════════════════════════════════

BEFORE (Hardcoded colors):
─────────────────────────
class PlayerCard extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Container(
      color: Color(0xFFE8F5F3),
      child: Text(
        'Player',
        style: TextStyle(color: Color(0xFF009688)),
      ),
    );
  }
}

AFTER (Using theme):
──────────────────
/// PLAYER MANAGEMENT SCREEN THEME
/// Primary: Teal | Secondary: Blue | Accent: Green
class PlayerCard extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Container(
      color: AppColors.playerCardBg,
      child: Text(
        'Player',
        style: TextStyle(color: AppColors.playerPrimary),
      ),
    );
  }
}

QUICK TIPS
═══════════════════════════════════════════════════════════════════════════════

💡 TIP 1: All colors are in ONE file (app_colors.dart)
   To change any color, just edit that file
   Everything automatically updates!

💡 TIP 2: Use screen-specific colors
   Each screen has its own primary/secondary/accent
   See QUICK_REFERENCE.dart for all assignments

💡 TIP 3: Use semantic colors for status
   paid → AppColors.paymentPaid (Green)
   pending → AppColors.paymentPending (Orange)
   failed → AppColors.paymentFailed (Red)

💡 TIP 4: Support dark mode with helpers
   AppColors.getBackgroundColor(isDark: isDark)
   AppColors.getTextColor(isDark: isDark)
   AppColors.getSurfaceColor(isDark: isDark)

💡 TIP 5: IDE auto-completion works!
   Type "AppColors." and your IDE shows all options
   No need to memorize anything
